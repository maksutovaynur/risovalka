# P1. Рисуем первый игровой мир

Цель урока: собрать первую игровую сцену, понять координаты, порядок рисования, цвета, картинки из файлов и первые кадры движения.

## 1. Подготовка

Нужно установить:

- Python 3.12 или новее;
- VS Code;
- расширения VS Code: Python, Pylance, Marp for VS Code.

На Windows при установке Python поставь галочку **Add python.exe to PATH**.

Открой папку проекта в VS Code. Если VS Code спросит про доверие к папке, выбери **Trust**. Открой любой пример, например `lessons/lesson1/samples/01_first_scene.py`, и выбери Python из `.venv`. Если окружения ещё нет, разреши VS Code создать его через `uv`.

## 2. Папка команды

Свой код и свои файлы сохраняй внутри папки команды:

```text
team/
  <team_name>/
    __init__.py
    README.md
    first_scene.py
```

В `README.md` напиши название команды, имена участников и короткое описание проекта.

Общие примеры лежат в `lessons/lesson1/samples/`. Их можно запускать и читать, но свой проект сохраняй в `team/<team_name>/`.

## 3. Первый запуск

Открой:

```text
lessons/lesson1/samples/01_first_scene.py
```

Запусти файл кнопкой запуска Python в VS Code. Должно появиться окно с первой сценой. Если окно кажется пустым, проверь, не открылось ли оно за VS Code.

## 4. Как менять пример

Меняй по одному месту и каждый раз запускай файл снова.

Сначала безопасно менять:

- координаты `x` и `y`;
- размеры фигур;
- цвета;
- текст;
- количество объектов;
- переменные вроде `hero_x` и `ground_y`;
- путь к картинке, если файл точно существует.

Не переименовывай команды вроде `draw_circle`, `clear_canvas`, `show_canvas`, пока не уверен, что понимаешь ошибку.

## 5. Что важно понять

| Термин | English в коде | Смысл |
| --- | --- | --- |
| координата | `x`, `y` | где рисовать объект |
| цвет | `RGB`, `HEX` | рецепт цвета |
| кадр | frame | одна картинка движения |
| пауза | `sleep` | время, чтобы увидеть кадр |
| вектор | `Vector` | сдвиг: куда и на сколько |
| точка | `Point` | место на экране |
| список | `list` | несколько значений подряд, например точки многоугольника |
| своя команда | `def function` | кусок кода с именем, который можно вызвать несколько раз |

Главные правила:

- `x` идёт вправо, `y` идёт вниз;
- компьютер выполняет команды сверху вниз;
- кто нарисован позже, тот виден сверху;
- новый кадр обычно начинается с очистки экрана;
- картинка из файла не загрузится, если путь или имя написаны неправильно.

## 6. Примеры урока

Открывай по порядку:

1. `01_first_scene.py` — первая сцена.
2. `02_color_recipes.py` — цвета.
3. `03_coordinate_map.py` — координаты.
4. `04_vectors.py` — вектор / `Vector`.
5. `05_manual_animation.py` — кадры и `sleep`.
6. `06_repeated_object.py` — повторяющийся код.
7. `07_example_use_function.py` — своя команда / `def function`.

## 7. Задания-поломки

Файлы в `lessons/lesson1/puzzles/` специально сломаны. Это не примеры для копирования.

Работай так:

1. Запусти задание.
2. Посмотри на ошибку или странный результат.
3. Исправь одну вещь.
4. Запусти снова.

В этих заданиях не всё про синтаксис. Часто нужно понять порядок команд, координаты, паузы, очистку экрана или путь к файлу.

## 8. Финальная работа

Собери в папке команды сцену на свою тему:

1. фон и минимум несколько объектов;
2. герой или предмет из картинки;
3. название игры;
4. понятные координаты;
5. короткое движение из нескольких кадров;
6. одно исправленное задание-поломка.

После этого отметь достижения в `achivements.md`.
