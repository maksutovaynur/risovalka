# Урок 1. Экран оживает

Практическая инструкция к занятию.

## Что получится

Сегодня ты создаёшь первую игровую сцену и небольшую покадровую анимацию: окно, фон, несколько объектов, заголовок и 3 кадра движения. Обязательная часть у всех похожая, а личные улучшения можно выбрать в `achivements.md`.

## 1. Установка на Windows

### Python

1. Открой https://www.python.org/downloads/
2. Скачай Python 3.12 или новее.
3. На первом экране установки поставь галочку **Add python.exe to PATH**.
4. Нажми **Install Now**.
5. Открой PowerShell и проверь:

```powershell
python --version
```

Если появилась версия Python, установка сработала.

### VS Code

1. Открой https://code.visualstudio.com/
2. Установи VS Code.
3. Открой вкладку расширений.
4. Установи расширения:
   - Python;
   - Pylance;
   - Marp for VS Code.

### Проект

1. Открой папку проекта в VS Code: **File → Open Folder**.
2. Открой терминал: **Terminal → New Terminal**.
3. Выполни команду:

```powershell
python -m pip install pyglet
```

Если преподаватель дал готовую папку с окружением, этот шаг, скорее всего, уже сделан.

## 2. Коротко для macOS и Linux

1. Установи Python 3.12 или новее.
2. Установи VS Code и расширения Python, Pylance, Marp for VS Code.
3. Открой папку проекта.
4. В терминале выполни:

```bash
python3 -m pip install pyglet
```

Если команда `python3` не работает, спроси преподавателя: на разных системах Python может называться по-разному.

## 3. Папка команды

Обычно команда состоит из двух учеников. Перед первой программой создайте отдельную папку внутри `team/`.

Структура:

```text
team/
  <team_name>/
    __init__.py
    README.md
    <project1_name>.py
    ... .py
```

Пример:

```text
team/
  space_cats/
    __init__.py
    README.md
    first_scene.py
    moon_animation.py
```

Что написать в `README.md`:

- название команды;
- имена участников;
- короткое описание команды;
- позже — описание проектов.

Правило проекта: весь код, картинки, звуки и другие файлы, которые создаёт команда, лежат внутри папки команды. Если файл нужен только вашей команде, он не должен лежать в общей папке урока.

Общие примеры остаются в `lessons/lesson1/samples/`. Их можно читать и запускать, но свой проект сохраняйте в `team/<team_name>/`.

## 4. Первый запуск

Открой файл:

```text
lessons/lesson1/samples/01_first_scene.py
```

Запусти его из терминала:

```powershell
python lessons/lesson1/samples/01_first_scene.py
```

Что должно получиться: откроется окно с первой игровой сценой. Окно останется открытым, пока ты сам его не закроешь.

Если окно появилось, но кажется пустым, не закрывай сразу: иногда окно открывается за VS Code.

## 5. Как работать с кодом

- Цветной текст в редакторе называется подсветкой синтаксиса. Она помогает различать числа, строки, имена и команды.
- Автодополнение подсказывает имена команд. Начни писать `game.` и посмотри список.
- После изменения файла нажми `Ctrl+S`.
- Чтобы увидеть новый результат, закрой окно программы и запусти файл ещё раз.
- Меняй по одному месту за раз. Так легче понять, что именно повлияло на картинку.

## 6. Что можно менять безопасно

В образцах сначала меняй:

- числа координат `x` и `y`;
- размеры фигур;
- названия цветов;
- текст заголовка;
- количество объектов;
- переменные вроде `hero_x` или `ground_y`.

Не меняй сразу названия команд `draw_circle`, `clear_canvas`, `show_canvas`: если в имени будет ошибка, Python не поймёт команду.

## 7. Карта понятий

| Понятие | Простое объяснение |
| --- | --- |
| Пиксель | Маленькая точка экрана |
| Растр | Картинка из пикселей |
| Цвет RGB | Смесь красного, зелёного и синего |
| Координаты | Адрес места на экране |
| Вектор | Сдвиг: куда и на сколько |
| Переменная | Имя для значения |
| Функция | Своя команда или готовая команда |

## 8. Частые ошибки Python

### SyntaxError

Python сообщает: «Я не понял запись». Часто не хватает скобки, кавычки или запятой.

Что делать: посмотри на строку из сообщения и строку прямо перед ней.

### NameError

Python сообщает: «Такого имени я не знаю». Например, было `hero_x`, а написали `herо_x` или `hero`.

Что делать: сравни имя в ошибке с тем местом, где переменная создаётся.

### IndentationError

Python сообщает: «С отступами что-то не так». Отступы показывают, какие строки относятся к одному блоку.

Что делать: проверь пробелы в начале строк. В одном блоке отступ должен быть одинаковым.

### TypeError

Python сообщает: «Команда получила не то, что ожидала». Например, для круга забыли указать радиус.

Что делать: проверь, сколько значений передано в команду.

### ValueError

Python сообщает: «Я понял, что это значение, но оно не подходит». Например, цвет `skyblue` не входит в список известных имён.

Что делать: используй знакомый цвет или HEX-рецепт вроде `#44aaff`.

### Файл не найден

Если картинка не загрузилась, путь к файлу может быть неправильным.

Что делать: проверь папку, расширение файла и пробелы в имени.

### Регистр букв

`Hero.png` и `hero.png` могут быть разными именами.

Что делать: копируй имя файла точно.

## 9. Задания-поломки

Файлы в `lessons/lesson1/puzzles/` специально ведут себя неправильно. Это не примеры для копирования, а маленькие площадки для поиска и исправления ошибок.

Правило:

1. Запусти задание.
2. Посмотри на ошибку или странный результат.
3. Найди место, где программа сбилась.
4. Исправь одну вещь.
5. Запусти снова.

Не пытайся чинить всё сразу.

## 10. Финальная сборка проекта

1. Создай папку команды в `team/`.
2. Возьми за основу идею из `lessons/lesson1/samples/01_first_scene.py` и перенеси её в свой файл внутри папки команды.
3. Измени фон и цвета.
4. Переставь объекты по координатам.
5. Добавь свой заголовок.
6. Открой `lessons/lesson1/samples/05_manual_animation.py`.
7. Сделай 3 кадра небольшого движения в файле своей команды.
8. Если добавляешь свои картинки или другие файлы, положи их в папку команды.
9. Отметь достижения в `achivements.md`.
