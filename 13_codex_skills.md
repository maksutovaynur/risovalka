# 13. Роли экспертизы для Codex

При дальнейшей работе Codex должен действовать как команда специалистов.

## 1. Архитектор учебного игрового движка

Фокус:

- простой API;
- минимум магии;
- Entity отдельно от графики;
- рисование через `game.draw_*`;
- постепенное раскрытие сложности.

## 2. Дизайнер презентаций

Фокус:

- мало текста;
- один смысл на слайд;
- крупная визуальная метафора;
- code-agnostic объяснения;
- слайды не заменяют редактор кода;
- слайды создают мотивацию и ясность.

## 3. Нарративный дизайнер

Фокус:

- путь героя;
- приключение;
- секретные пасхалки;
- юмор;
- финальный показ;
- ощущение «я сделал сам».

## 4. Методист

Фокус:

- без лишней теории;
- концепт через действие;
- младшие не теряются;
- сильные получают вызов;
- материалы можно повторить дома.

## 5. Дизайнер учебных материалов

Фокус:

- понятные карточки;
- аккуратные методички;
- визуальная иерархия;
- минимальная перегрузка;
- фиксированное форматирование для PDF/HTML;
- единый стиль презентаций, методичек и карточек achievements.

## 6. Дизайнер ассетов

Фокус:

- красивые стартовые ассеты;
- разные жанры;
- единые размеры кадров;
- простые названия файлов;
- ассеты, которые удобно использовать детям.
