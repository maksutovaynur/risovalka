# 09. Принципы качества курса

Этот файл — чеклист для всех материалов: презентаций, методичек, игровых примеров, движка и заданий.

## 1. Без лишней теории раньше времени

Теория появляется только тогда, когда она нужна для ближайшего действия.

Плохо:

> «Сегодня изучаем циклы, условия и массивы».

Хорошо:

> «Чтобы не рисовать 100 кадров руками, нам нужен цикл».

## 2. Концепты через необходимость что-то реализовать

Каждая абстракция должна отвечать на вопрос: «Как мне сделать это в игре?»

| Концепт | Через какую необходимость вводится |
|---|---|
| Координаты | Поставить героя в нужное место |
| Цвета | Сделать мир своим |
| Переменная | Запомнить позицию, скорость, очки |
| Условие | Реакция на кнопку или столкновение |
| Цикл | Повторять кадры |
| Список | Много монет/врагов/платформ |
| Функция | Не повторять одно и то же |
| Entity | Объекту нужны свойства |
| Sprite | Удобно менять кадры анимации |
| dt | Двигаться честно по времени |

## 3. Code-agnostic объяснения

В презентациях концепты программирования объясняются независимо от языка.

Примеры:

- условие — развилка: «если дождь, взять зонт»;
- цикл — повтор: «пока песня играет, танцуем»;
- список — коробка с карточками/монетами;
- переменная — подписанная коробочка;
- функция — своя команда/рецепт.

Язык программирования появляется в редакторе, когда нужно реализовать механику.

## 4. Весёлые хуки

Ошибки используются как шоу:

- объект исчез из-за очистки;
- герой оставляет следы;
- герой улетел за экран;
- враг слишком быстрый;
- все монеты появились в одном месте;
- текст стал гигантским;
- счёт ушёл в минус;
- игра стала «порталом хаоса».

## 5. Пасхалки и секреты

В каждой сессии нужны секретные карточки:

- секретный эффект;
- спрятанный объект;
- кодовая фраза;
- смешной режим;
- конфетти;
- портал;
- огонь;
- шейдер;
- режим «ночь»;
- второй игрок.

## 6. Интерактивные achievements

Achievements должны ощущаться как путь героя, а не как контрольная работа.

Нужны:

- дерево навыков;
- лидерборд на проекторе;
- badges за творчество;
- badges за помощь другим;
- badges за смешную ошибку;
- финальный показ.

## 7. Ощущение авторства

Каждый ребёнок должен иметь момент:

> «Это сделал я».

Для этого нужны:

- выбор темы;
- выбор ассетов;
- возможность принести свои ассеты;
- свобода в правилах игры;
- финальный показ;
- возможность забрать проект домой;
- возможность собрать/показать игру вне курса.
