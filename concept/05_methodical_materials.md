# 05. Методические материалы

## Главный принцип

Материалы должны позволять ребёнку повторить занятие дома самостоятельно, даже если он пропустил сессию.

Каждый экранный материал преподавателя должен иметь детскую версию в папке материалов.

## Форматы

Исходники:

- `.md` или `.qmd` для методичек;
- `.marp.md` для презентаций;
- `.py` для демонстрационных примеров;
- `.json`/`.csv` для achievements/leaderboard, если потребуется.

Финальные версии для детей:

- PDF;
- HTML;
- папка проекта, которую можно открыть в VS Code.

## Структура материалов сессии

Для каждой сессии:

```text
sessions/session_N/
  README.md
  slides/
  student_workbook.md
  teacher_notes.md
  achievements.md
  start_project/
  examples/
  bug_showcase/
  assets/
  secret_cards/
```

## Методичка ученика

Структура:

1. Что сегодня делаем.
2. Что получится в конце.
3. Мини-карта понятий.
4. Шаги практики.
5. Achievements.
6. Частые ошибки.
7. Секретные задания.
8. Что можно улучшить дома.

## Памятка по ошибкам Python

Нужна отдельная памятка: `student_materials/python_errors_cheatsheet.md`.

Принцип: не «ошибки страшные», а «компьютер объясняет, где запутался».

Обязательные ошибки:

- `SyntaxError` — компьютер не понял фразу;
- `NameError` — такого имени нет;
- `FileNotFoundError` — не найдена картинка/файл;
- `IndentationError` — проблема с отступами;
- `TypeError` — команда получила не то, что ожидала;
- ошибка русской/английской буквы в имени файла;
- ошибка регистра `Hero.png` vs `hero.png`.

## Хуки и курьёзы

В каждой сессии должен быть блок «Баг дня»:

- объект исчез, потому что экран очистили после рисования;
- герой оставляет следы, потому что забыли очистку;
- герой улетел за экран;
- враг слишком быстрый;
- счёт стал отрицательным;
- все монеты появились в одном месте;
- картинка не загрузилась из-за пробела/кириллицы в имени файла.

## Принцип визуального материала

- мало текста;
- один смысл на слайд;
- крупная картинка/метафора;
- код — в редакторе, не на слайде;
- если код на слайде нужен, то только псевдокод или 3–5 строк.

## Материалы для преподавателя

Должны включать:

- сценарий занятия по минутам;
- какие примеры открыть;
- какие ошибки показать;
- что делать, если группа идёт медленно;
- что дать сильным;
- какие секретные функции можно раскрыть;
- как фиксировать achievements;
- как провести финальный показ.
