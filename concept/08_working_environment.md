# 08. Рабочее окружение

## Цель

Среда должна одинаково ощущаться на Linux у преподавателя и на Windows 11 у ребят.

Принцип:

> Дети открывают папку в VS Code и нажимают Run. Настройка спрятана.

## Основной стек

- VS Code;
- Python;
- Python Extension;
- uv;
- готовые `.vscode` конфигурации;
- Marp for VS Code;
- Markdown/Quarto для методичек;
- готовые скрипты запуска;
- готовый шаблон проекта.

## Инструкция для преподавателя

1. Установить VS Code.
2. Установить Python.
3. Установить uv.
4. Открыть папку курса в VS Code.
5. Установить рекомендуемые расширения.
6. Проверить запуск примеров через VS Code tasks.
7. Проверить Marp preview.
8. Проверить экспорт презентаций.

## Инструкция для детей

1. Открыть VS Code.
2. Открыть папку занятия.
3. Открыть `main.py`.
4. Нажать Run или выбрать задачу `Run current session`.
5. Не настраивать интерпретатор руками, если всё уже подготовлено.

## Что должно быть готово заранее

- Python установлен;
- VS Code установлен;
- расширения установлены;
- зависимости установлены через uv;
- папки ассетов на месте;
- стартовые проекты открываются;
- запуск работает без терминального шаманства;
- презентации открываются в VS Code.

## Зачем uv

`uv` нужен как инженерная основа, но не как тема для детей.

Для преподавателя и Codex:

- единая установка зависимостей;
- воспроизводимость;
- запуск задач;
- сборка exe;
- одинаковая структура проекта.

Для детей:

- uv спрятан за кнопками/скриптами.

## VS Code конфигурации

Нужны файлы:

- `.vscode/extensions.json`;
- `.vscode/settings.json`;
- `.vscode/tasks.json`;
- `.vscode/launch.json`.

Они должны быть готовы в репозитории.
