# 03. Концепция учебной Python-библиотеки

## Цель библиотеки

Библиотека `gamekit` должна позволить детям быстро создавать 2D-игры, не погружаясь в технический стек, но сохраняя честную модель программирования.

Она не должна быть визуальным конструктором. Она должна быть ближе к PyGame, но с педагогически простым API.

## Главные принципы

### 1. Рисование всегда через `game.draw_*`

Игровые объекты не рисуют себя сами.

Модель:

- `game` — художник;
- `Entity` — игровой объект с координатами и свойствами;
- изображение/спрайт/фигура — способ показать объект.

### 2. Логика и графика разделены

Герой не обязан быть картинкой. Его можно нарисовать как:

- картинку;
- спрайт;
- круг;
- прямоугольник;
- текст;
- невидимую зону;
- коллайдер.

### 3. Entity — удобный мешочек игровых свойств

Основная модель для детей:

- `x`, `y`;
- `angle`;
- `speed`;
- `health`, `mana`, `score`, `name` и другие пользовательские свойства.

Дети могут добавлять свойства объекту без изучения классов.

### 4. Движение не должно выглядеть магией

Можно использовать удобные методы вроде `ship.move_forward()`, но в материалах нужно показывать, что в этот момент меняются координаты.

Обязательные демонстрации:

- вывести координаты до движения;
- выполнить движение;
- вывести координаты после движения;
- показать координаты на экране рядом с объектом.

### 5. Body можно использовать внутри, но не показывать рано

Внутри библиотеки можно иметь `Body` для позиции, поворота, скорости и коллизий. В детском API лучше давать прямой интерфейс:

- `entity.x`;
- `entity.y`;
- `entity.angle`;
- `entity.speed`;
- `entity.move()`;
- `entity.rotate()`;
- `entity.move_forward()`.

Отдельный `Body` можно раскрыть только как продвинутую тему.

## Минимальное API движка

- создание окна;
- очистка экрана;
- вывод кадра;
- `sleep` для ручной анимации;
- рисование круга, прямоугольника, линии, текста;
- загрузка и рисование изображения;
- загрузка и рисование спрайта;
- работа с клавиатурой;
- работа с мышью (*);
- Entity;
- движение Entity;
- вращение Entity;
- движение вперёд по направлению;
- простые столкновения;
- камера (*);
- тайлы/замощение фоном (*);
- эффекты/пасхалки (*).

## Секретные функции для вау-эффекта

В библиотеке нужно иметь больше функций, чем раскрывается в базовом курсе.

Примеры:

- огонь;
- дождь;
- снег;
- портал;
- свечение;
- след за персонажем;
- экранная тряска;
- конфетти победы;
- параллакс;
- пиксельный фильтр;
- «режим хакера»;
- звёздное небо;
- магическая аура.

Эти функции нужны не для обязательной программы, а для азарта: ребёнок видит, что можно открыть «секретный приём» и сделать игру эффектнее.
