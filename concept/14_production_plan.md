# 14. План производства материалов

## Цель

После скачивания папки курса можно сразу запускать Codex и двигаться по конкретным задачам.

## Шаг 1. Закрепить инструменты

- Marp Markdown для презентаций.
- Markdown/Quarto для методичек.
- VS Code как единая рабочая среда.
- Python + uv для проекта.
- Готовые `.vscode` задачи.

## Шаг 2. Доработать презентации

Приоритет:

1. P0 — вводная мотивационная.
2. P1 — графика и первые кадры.
3. P2 — игровой цикл и ввод.
4. P3 — правила, враги, AI.
5. P4 — финальный проект и показ.

## Шаг 3. Реализовать минимальный `gamekit`

Сначала только то, что нужно для первой сессии:

- окно;
- очистка;
- рисование фигур;
- текст;
- картинки;
- sleep;
- next_frame;
- простые ассеты.

Потом:

- игровой цикл;
- клавиатура;
- мышь;
- Entity;
- спрайты;
- столкновения;
- камера;
- эффекты.

## Шаг 4. Создать примеры

Для каждой сессии:

- start project;
- final example;
- teacher demo;
- bug showcase;
- secret example;
- advanced example.

## Шаг 5. Создать методички

Для каждой сессии:

- student workbook;
- teacher notes;
- achievements;
- cheat sheet;
- home replay guide.

## Шаг 6. Создать ассеты

Сначала универсальные:

- герой;
- враг;
- монета;
- фон;
- платформа;
- кнопка;
- портал;
- эффект победы.

Потом жанровые наборы.

## Шаг 7. Создать вспомогательные инструменты

Минимум:

- achievements/leaderboard;
- bitmap editor;
- sprite editor;
- vector editor.

## Definition of Done для материалов

Материал готов, если:

- открывается в VS Code;
- понятен без преподавателя хотя бы частично;
- есть базовый и продвинутый уровень;
- есть смешной баг/хук;
- есть секретная карточка;
- есть понятный результат «что показать»;
- можно повторить дома.
