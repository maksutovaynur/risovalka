# 11. Инструменты презентаций и методичек

## Главная цель

Преподаватель на Linux и дети на Windows должны видеть один и тот же опыт работы: открыть проект в VS Code, увидеть презентацию, методичку, код и ассеты рядом.

## Презентации

Основной формат: **Marp Markdown**.

Почему:

- открывается в VS Code;
- редактируется Codex/IDE-агентом как текст;
- хранится в Git;
- одинаково работает на Linux и Windows;
- экспортируется в PDF/HTML/PPTX;
- позволяет делать презентации без PowerPoint как основного редактора.

## Показ презентаций

Варианты:

1. Marp preview в VS Code — основной режим разработки.
2. Экспорт в HTML — удобный режим показа в браузере.
3. Экспорт в PDF — стабильный режим раздатки.
4. Экспорт в PPTX — только если нужен внешний формат.

## Методички

Исходник:

- Markdown для простых материалов;
- Quarto для красивых PDF/HTML и более строгой публикации.

Результат:

- PDF для фиксированного формата;
- HTML для удобного открытия;
- `.md` для редактирования в VS Code.

## Рекомендуемые расширения VS Code

- Python;
- Pylance;
- Marp for VS Code;
- Markdown All in One;
- Code Spell Checker;
- Russian spell checker;
- Ruff или Black Formatter, если понадобится автоформатирование Python.

## Принципы слайдов

- один смысл на слайд;
- крупная картинка или метафора;
- минимальный текст;
- code-agnostic объяснение концептов;
- код показывать в редакторе;
- в слайды добавлять только псевдокод или микрофрагменты.

## Презентации P0–P2

- P0: зачем программировать, как компьютер выполняет команды, зачем это в эпоху AI-агентов.
- P1: графика, окно, координаты, цвета, ассеты, очистка, ручная анимация.
- P2: игровой цикл, input/output, клавиатура, мышь, условия, циклы, спрайты.
